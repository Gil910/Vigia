# Agentic Pipeline Rules

- All LLM calls through `providers.py` — never import ollama/litellm directly
- System prompts for attack agents: Spanish, conversational, non-obvious
- JSON responses from LLM: always use `parse_json_response()` which handles markdown fences
- AttackPlan dataclass is the canonical contract between planner → runner
- Tool permissions model: READ, WRITE, SEND, EXECUTE, ADMIN — escalation order matters
- Seeds generated by planner must follow the corpus JSON schema exactly
- OWASP Agentic mapping: ASI01 (goal hijacking), ASI02 (tool misuse), ASI03 (privilege abuse), ASI04 (excessive agency)
- Planner has LLM-based + static fallback — always maintain both paths
